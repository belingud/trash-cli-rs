# rm 选项兼容性

本文档是 `trash-cli-rs` 的公共兼容性契约。

## 范围

- 主要用例：别名，如 `alias rm="trash"`。
- 主要操作：将给定路径移动到系统回收站。
- 解析规则：文档化的兼容性标志仅在精确的 `--` 之前被忽略。
- 字面规则：在 `--` 之后，每个后续参数都被视为字面操作数。
- 安全规则：未知的虚线参数不会被静默吞下；它们被视为操作数。
- 诊断规则：缺失路径的报错会刻意使用带 `trash` 前缀的文案，例如 `trash: <path>: No such file or directory`。

## 长期兼容性承诺

- 已文档化的忽略参数，不会在补丁版本里静默改变行为。
- 不支持的虚线参数，会继续按字面操作数处理；如果未来真的要改，只会在明确记录的未来 major 版本中作为 breaking change 处理。
- 参数解析规则目标上保持平台无关；后端差异应记录在 [platform-differences_zh.md](platform-differences_zh.md) 中。

## 非目标

`trash-cli-rs` 不尝试复制 `rm` 语义。
忽略标志仅意味着"在删除剩余操作数时接受它而不报错"。
这并不意味着该工具在该标志后面实现了原始的 `rm` 行为。

该工具不承诺：

- 强制删除行为
- 交互式确认提示
- 带有每个条目逻辑的递归目录遍历
- 安全覆盖/粉碎
- 白名单取消删除行为
- 挂载点遍历控制

## 内置选项

这些是 `trash-cli-rs` 选项，而不是 `rm` 兼容性选项。

| 选项              | 状态 | 行为                                             |
| ----------------- | ---- | ------------------------------------------------ |
| `-h`, `--help`    | 支持 | 显示帮助文本并退出 `0`。                         |
| `-V`, `--version` | 支持 | 显示版本并退出 `0`。                             |
| `--`              | 支持 | 停止解析兼容性标志；所有后续参数都是字面操作数。 |

## macOS / BSD rm 短选项

参考方言：macOS `/bin/rm` (`rm [-f | -i] [-dIRrvWx] file ...`)。

| 选项 | rm 含义                  | `trash-cli-rs` 状态 | `trash-cli-rs` 中的实际行为                    |
| ---- | ------------------------ | ------------------- | ---------------------------------------------- |
| `-d` | 删除目录                 | 忽略                | 目录操作数仍作为路径传递给回收站。             |
| `-f` | 强制，抑制缺少文件诊断   | 忽略                | 缺少的路径仍会产生错误。                       |
| `-i` | 每次删除前提示           | 忽略                | 不显示提示。                                   |
| `-I` | 对危险删除提示一次       | 忽略                | 不显示提示。                                   |
| `-P` | macOS 上的向后兼容无操作 | 忽略                | 被接受为无操作兼容性标志。                     |
| `-R` | 递归删除                 | 忽略                | 不实现递归遍历；操作数路径作为单个路径被删除。 |
| `-r` | 与 `-R` 相同             | 忽略                | 与 `-R` 行为相同。                             |
| `-v` | 详细输出                 | 忽略                | 成功保持静默。                                 |
| `-W` | 尝试对白名单取消删除     | 不支持              | 被视为字面操作数。                             |
| `-x` | 不跨越挂载点             | 忽略                | 被接受为无操作兼容性标志。                     |

## GNU rm 长选项兼容性

接受这些选项是因为许多 `rm` 别名和 shell 习惯来自 GNU/Linux。

| 选项                   | `trash-cli-rs` 状态 | `trash-cli-rs` 中的实际行为          |
| ---------------------- | ------------------- | ------------------------------------ |
| `--dir`                | 忽略                | 除了删除操作数路径外，没有特殊语义。 |
| `--force`              | 忽略                | 缺少的路径仍会产生错误。             |
| `--interactive`        | 忽略                | 不显示提示。                         |
| `--interactive=always` | 忽略                | 不显示提示。                         |
| `--interactive=never`  | 忽略                | 缺少的路径仍会产生错误。             |
| `--interactive=once`   | 忽略                | 不显示提示。                         |
| `--recursive`          | 忽略                | 不实现递归遍历。                     |
| `--verbose`            | 忽略                | 成功保持静默。                       |
| `--one-file-system`    | 忽略                | 不实现挂载点遍历逻辑。               |
| `--preserve-root`      | 忽略                | CLI 层不实现根保护逻辑。             |
| `--preserve-root=all`  | 忽略                | 与 `--preserve-root` 相同。          |
| `--no-preserve-root`   | 忽略                | 与 `--preserve-root` 相同。          |

## 短选项集群

如果短选项集群中的每个字母都是文档化的兼容性标志，则整个集群被忽略。

示例：

- `-rf`
- `-rfxP`
- `-Riv`

如果集群包含任何不支持的字母，则整个标记被视为字面操作数。

示例：

- `-W`
- `-rW`
- `-rxW`

## 前导连字符文件名

要删除以 `-` 开头的文件名，请显式消除歧义：

```bash
trash -- -rf
trash ./-rf
trash /tmp/-rf
```

仅引用是不够的，因为 shell 在构造 `argv` 之前会移除引号。
